⏱ バッチ実行条件一覧

曜日ごと・cron実行順　（実際の crontab をもとに作成）

土曜日 / Saturday
05:00
SummaryRacesPopularityRatio.php
race_result_history の全データ（過去全期間）をもとに、レースごとの「人気順オッズの連続比率」を計算してマスタテーブルに保存する
読むt_horse_odds_finder_race_result_history（日付絞りなし・全件。popularity_rank 昇順で tan を取得）
書くt_horse_odds_finder_races_popularity_ratio（EXISTSチェック → 未登録レースのみ INSERT）
⚠️ 既に登録済みのレースはスキップ（冪等）。このマスタが 08:00 の ImportRacesPopularityRatio の比較元になる。
05:50
deleteKeibaTableRecords.php
今週のレース開始前に運用テーブル7本を TRUNCATEしてリセットする
書くTRUNCATE：horses / odds / odds_get_timing / odds_wide / race_results / races / schedules
⚠️ 06:00 の importSchedule より必ず先に実行すること。累積テーブル（race_result_history 等）は対象外。
06:00
ImportKeibaSchedule.php
土・日 両日分の開催スケジュール・レース・馬情報を JRA サイトから取得して保存する
書くt_horse_odds_finder_schedules / races / horses（INSERT）
土曜は 05:50 の TRUNCATE 済みのため odds の DELETE 不要。毎日 cron だが土日以外は実質空振り。
07:00
ImportKeibaBaseOdds.php
土・日 両日分のオッズを初回一括取得する（cron: 0 7 * * 6）
書くt_horse_odds_finder_odds（minutes_before_start=999 でUPSERT）
書くt_horse_odds_finder_odds_get_timing（タイミング記録を UPSERT）
WHERE date >= 今日（土曜）で取得するため、土・日 両日分のオッズが一度に保存される。リトライ最大3回。
08:00
ImportRacesPopularityRatio.php
当日（土曜）のレースの人気比率を 05:00 で作成したマスタと照合し、RMSE 類似度を算出してレーステーブルに保存する
読むt_horse_odds_finder_races_popularity_ratio（マスタ・全件をメモリ展開）
読むt_horse_odds_finder_races（WHERE popularity_ratio IS NULL）
読むt_horse_odds_finder_odds（fallback: ODDS_GET_TIMING 順でオッズ取得）
書くt_horse_odds_finder_races.popularity_ratio / popularity_ratio_table_ids / popularity_ratio_match_percent を UPDATE
matchPercent ≥ 70% のマスタIDを '|' 区切りで保存。オッズが1件もない場合はスキップ。
09〜17時毎分
ImportKeibaOdds.php
発走タイミング（残り 24/21/18/15/12/9/6/3/0 分）に合致するレースのみオッズをリアルタイム取得する（flock）
読むt_horse_odds_finder_races（当日・未発走レース一覧）
読むt_horse_odds_finder_odds（前回オッズとの比較用）
書くt_horse_odds_finder_odds（UPSERT）
書くt_horse_odds_finder_odds_get_timing（タイミング記録 INSERT/UPDATE）
人気順位の変化 ≥ 2 の場合に WebPush 送信。diff=24→999、diff=0→-999 で保存。flock で多重起動防止。
09〜18時毎分
ImportKeibaJraRaceOneResult.php
レース終了後すぐ（発走+10/15/20/25/30分）にレース結果を取得する（flock）
書くt_horse_odds_finder_race_results（INSERT。notified_at も設定）
$existingKeys チェックで重複 INSERT を防止。結果確定時に WebPush 送信。
18:00
SummaryKeibaInfo.php
当日のレース情報をもとにサマリを作成する。同時に race_result_history への UPSERT も実行する（確定オッズ使用）
読むt_horse_odds_finder_horses / races / odds（WHERE minutes_before_start=-999 確定オッズ）
読むt_horse_odds_finder_summary（重複チェック用既存キー）
読むt_horse_odds_finder_race_result_history（既存 popularity_rank を保持するため参照）
書くt_horse_odds_finder_summary（INSERT。重複はスキップ）
書くt_horse_odds_finder_race_result_history（UPSERT：basho / race_name / name / tan / fuku_min / fuku_max / popularity_rank を保存）
★重要：この UPSERT が先行するため、19:00 の ImportKeibaRaceResultHistory は全開催が PRE-SKIP となり「頭数:0 / 開催:N」通知になる。これは正常動作。
18:30
ImportKeibaJraRaceResult.php
当日のレース着順（result）を JRA サイトから一括取得し、サマリテーブルに書き込む
書くt_horse_odds_finder_summary.result（着順カラムのみ UPDATE。WHERE result IS NULL で未確定のみ対象）
※ race_results テーブルではなく summary テーブルの result カラムを更新する。BASHO_MAP で漢字→コード変換。
19:00
ImportKeibaRaceResultHistory.php
当日（土曜）のレース結果を Node.js 経由で JRA サイトから取得し race_result_history に保存する
書くt_horse_odds_finder_race_result_history（INSERT）
⚠️ 18:00 の SummaryKeibaInfo が全開催を先に UPSERT 済みのため、ほぼ全開催が PRE-SKIP となる（新規保存=0件が正常）。バックアップ・補完として機能。
20:00
SummaryHistoryPopularityRank.php
race_result_history の人気順（popularity_rank）を tan 昇順で算出し更新する
読むt_horse_odds_finder_race_result_history（WHERE date BETWEEN 当月1日〜末日）
書くt_horse_odds_finder_race_result_history.popularity_rank を UPDATE
全馬に popularity_rank が既に入っているレースはスキップ。NULL/非数値の tan は最後尾に並べる。
20:30
SummaryPopularityRankAverage.php
🚫 土曜日は cron 対象外のため実行なし（cron: 30 20 * * 0-5）
21:00
SummaryHistoryFinishingPosition.php

読むt_horse_odds_finder_race_result_history（WHERE date LIKE '当月%' AND finishing_position IS NULL OR = 0）
書くt_horse_odds_finder_race_result_history.finishing_position を UPDATE
非入賞（競走中止等）は -1 を格納。以降の処理でスキップ対象になる。
21:30

券種（単勝/複勝/枠連/ワイド/馬連/馬単/3連複/3連単）ごとに '馬番|金額' 形式で保存。複数は '/' で結合。
