Claude Code へ — このドキュメントについて

これは Web Push 通知機能の汎用実装ガイドです。
赤文字の部分が「プロジェクト固有の名前（要置換）」です。
次のプロジェクトで実装するときは、赤文字を実際のプロジェクト名に置き換えてください。

このガイドは Flutter web（Dart）＋ Laravel で動作確認済みの構成です。
Firebase は一切使いません。Web Push Protocol（VAPID 方式）のみです。

Web Push 通知 完全実装ガイド

★ 赤文字 = 暫定プロジェクト名「BlueSky」。実際のプロジェクト名に置き換えること。

Firebase を一切使わず、ブラウザ標準の Web Push Protocol（VAPID 方式） で実装する。
Laravel バックエンドが VAPID 認証で FCM（Chrome）/ Apple Push（Safari）サーバー経由でブラウザに通知を届ける。
フロントエンドは Flutter web（Dart）。Service Worker で push イベントを受信して通知を表示する。

1. 全体アーキテクチャ

【通知登録フロー】 ブラウザ（スマホ / PC） ┌──────────────────────────────────────────────────────────┐ │ アプリ起動 or ログイン成功 │ │ ↓ │ │ FcmService.registerToken(userId) │ │ ↓ │ │ GET https://（ドメイン）/api/vapid-public-key │ │ ↓ 公開鍵を受け取る │ │ index.html: subscribeWebPush(vapidPublicKey) │ │ ↓ │ │ push-sw.js を scope=/blue_sky/push/ で Service Worker 登録 │ │ ↓ SW が activated になるまで待機 │ │ pushManager.subscribe() → Subscription 取得 │ │ { endpoint, keys: { p256dh, auth } } │ │ ↓ │ │ POST https://（ドメイン）/api/web-push/subscribe │ │ ↓ DB に upsert 保存（endpoint をキーに重複防止） │ └──────────────────────────────────────────────────────────┘ 【通知送信フロー】 Laravel（cron / artisan コマンド） ↓ BlueSkyWebPushService::sendPushDeveloperNews() ← 開発者宛のみ BlueSkyWebPushService::sendPushOddsNews() ← 全ユーザー宛 ↓ DB から subscription レコードを取得 ↓ minishlink/web-push（VAPID 認証） ↓ FCM サーバー（Chrome）/ Apple Push サーバー（Safari） ↓ push-sw.js の push イベント → showNotification() ↓ デバイスに通知表示

DB テーブル構成

カラム型説明
idINT AUTO_INCREMENT PK自動採番
user_idVARCHAR(255)ログインユーザーID
endpointTEXT / UNIQUEFCM / Apple の push エンドポイント URL（端末×ブラウザで一意）
p256dhVARCHAR(512)暗号化キー
authVARCHAR(255)認証シークレット

endpoint の特性： 同じ端末のブラウザでキャッシュをクリアすると endpoint が変わり、新しいレコードが INSERT される。
古い（無効な）レコードは次回の push 送信時に 410 Gone が返り、自動で DELETE される。
1ユーザーが複数端末・複数ブラウザを使うと、その数だけレコードが存在するのが正常。

2. 最重要：Flutter SW との競合問題

ここを理解せずに実装すると、スマホで通知が届かない。

何が起きるか

Flutter web は flutter_service_worker.js を自動生成する。
このファイルには self.skipWaiting() が含まれており、インストールされると即座に active になってスコープを支配する。

push-sw.js を 同じスコープ（例: /blue_sky/）に登録すると：

  1. flutter_service_worker.js が active（支配者）
  2. push-sw.js が登録される → installing → waiting（順番待ち）
  3. push-sw.js は 永遠に activated にならない（Flutter SW が居座るため）
  4. push イベントは active な SW（= flutter_service_worker.js）に届く
  5. flutter_service_worker.js には push ハンドラがない → 通知が表示されない
解決策：push-sw.js を Flutter SW とは別スコープに登録する。
{ scope: '/blue_sky/push/' } を指定することで Flutter SW と干渉しない独立した登録になり、 push-sw.js はすぐに activated になって push イベントを正常に受け取れる。 
// ❌ NG: Flutter SW と同じスコープ
navigator.serviceWorker.register('/blue_sky/push-sw.js', {
  scope: '/blue_sky/'
});

// ✅ OK: /push/ を付けた別スコープ
navigator.serviceWorker.register('/blue_sky/push-sw.js', {
  scope: '/blue_sky/push/'
});

5 WebPushService を作成

app/Services/BlueSkyWebPushService.php
<?php

namespace App\Services;

use Illuminate\Support\Facades\DB;
use Illuminate\Support\Facades\Log;
use Minishlink\WebPush\Subscription;
use Minishlink\WebPush\WebPush;

class BlueSkyWebPushService
{
    private WebPush $webPush;

    public function __construct()
    {
        $auth = [
            'VAPID' => [
                'subject'    => config('services.vapid.subject'),
                'publicKey'  => config('services.vapid.public_key'),
                'privateKey' => config('services.vapid.private_key'),
            ],
        ];
        $this->webPush = new WebPush($auth);
    }

    // 開発者向け通知（$developerUserIds に含まれるユーザーのみに送信）
    // 各 artisan コマンドの finally ブロックで呼んで実行ログをスマホで受け取る用途
    public function sendPushDeveloperNews(string $title, string $body): void
    {
        $developerUserIds = ['your_user_id']; // ← 開発者のユーザーID

        $subscriptions = DB::table('t_blue_sky_push_subscriptions')
            ->whereIn('user_id', $developerUserIds)
            ->get();

        $this->sendToSubscriptions($subscriptions, $title, $body);
    }

    // 全ユーザー向け通知（is_delete = 0 の全 subscription に送信）
    public function sendPushOddsNews(string $title, string $body): void
    {
        $subscriptions = DB::table('t_blue_sky_push_subscriptions')
            ->where('is_delete', '0')
            ->get();

        $this->sendToSubscriptions($subscriptions, $title, $body);
    }

    private function sendToSubscriptions($subscriptions, string $title, string $body): void
    {
        foreach ($subscriptions as $row) {
            $subscription = Subscription::create([
                'endpoint' => $row->endpoint,
                'keys'     => [
                    'p256dh' => $row->p256dh,
                    'auth'   => $row->auth,
                ],
            ]);

            $this->webPush->queueNotification(
                $subscription,
                json_encode([
                    'title' => $title,
                    'body'  => $body,
                    'icon'  => '/blue_sky/icons/Icon-192.png',
                    'url'   => 'https://（ドメイン）/blue_sky/',
                ])
            );
        }

        // flush() で実際に送信し、結果を1件ずつ確認する
        foreach ($this->webPush->flush() as $report) {
            if (!$report->isSuccess()) {
                Log::warning('WebPush送信失敗: ' . $report->getReason() . ' endpoint: ' . $report->getEndpoint());
                // 410 Gone = subscription が無効 → 自動削除
                DB::table('t_blue_sky_push_subscriptions')
                    ->where('endpoint', $report->getEndpoint())
                    ->delete();
            }
        }
    }
}

6 WebPushController を作成

app/Http/Controllers/Api/BlueSkyWebPushController.php
<?php

namespace App\Http\Controllers\Api;

use App\Http\Controllers\Controller;
use Illuminate\Http\JsonResponse;
use Illuminate\Http\Request;
use Illuminate\Support\Facades\DB;

class BlueSkyWebPushController extends Controller
{
    // Flutter から subscription 情報を受け取り DB に保存する
    // endpoint をキーにして upsert するので、同じ端末から何度登録しても重複しない
    public function subscribe(Request $request): JsonResponse
    {
        $request->validate([
            'user_id'  => 'required|string',
            'endpoint' => 'required|string',
            'p256dh'   => 'required|string',
            'auth'     => 'required|string',
        ]);

        DB::table('t_blue_sky_push_subscriptions')->upsert(
            [
                'user_id'   => $request->input('user_id'),
                'endpoint'  => $request->input('endpoint'),
                'p256dh'    => $request->input('p256dh'),
                'auth'      => $request->input('auth'),
                'is_delete' => 0,
            ],
            ['endpoint'],                               // 一意キー（このカラムで重複判定）
            ['user_id', 'p256dh', 'auth', 'is_delete'] // 重複時に更新するカラム
        );

        return response()->json(['success' => true]);
    }

    // Flutter が VAPID 公開鍵を取得するためのエンドポイント
    public function vapidPublicKey(): JsonResponse
    {
        return response()->json([
            'public_key' => config('services.vapid.public_key'),
        ]);
    }
}

// push イベント: バックグラウンドでも通知を表示する
self.addEventListener('push', function(event) {
  let data = {};
  if (event.data) {
    try {
      data = event.data.json();
    } catch (e) {
      data = { title: 'お知らせ', body: event.data.text() };
    }
  }
  const title   = data.title ?? 'お知らせ';
  const options = {
    body: data.body ?? '',
    icon: '/blue_sky/icons/Icon-192.png',
    data: { url: data.url ?? 'https://（ドメイン）/blue_sky/' },
  };
  event.waitUntil(self.registration.showNotification(title, options));
});

self.addEventListener('notificationclick', function(event) {

9 index.html に subscribeWebPush を追加

<head> 内の <script> タグとして追加する（flutter_bootstrap.js の読み込みより前に書くこと）。

scope は必ず /blue_sky/push/ にすること。
/blue_sky/ にすると Flutter SW と競合して通知が届かなくなる（「2. 最重要」参照）。
web/index.html の <head> 内（<script> タグとして追加）
<script>
  window.subscribeWebPush = async function(vapidPublicKey) {
    try {
      // 通知許可を求める
      const permission = await Notification.requestPermission();
      if (permission !== 'granted') return null;

      // push-sw.js を Flutter SW（flutter_service_worker.js）とは別スコープで登録する。
      // 同一スコープにすると Flutter SW の skipWaiting() に負けて
      // push-sw.js が waiting のまま activated にならず push イベントが届かない。
      const registration = await navigator.serviceWorker.register(
        '/blue_sky/push-sw.js',
        { scope: '/blue_sky/push/' }
      );

      // 既存の Subscription があればそのまま再利用して返す。
      // 毎回 unsubscribe すると再登録失敗時に DB 上に無効なエンドポイントが残る。
      const existing = await registration.pushManager.getSubscription();
      if (existing) {
        return JSON.stringify(existing);
      }

      // 新規登録時は SW が active になるまで待つ。
      // 別スコープなので Flutter SW と競合せず、すぐに activated になる。
      if (!registration.active) {
        await new Promise(resolve => {
          const sw = registration.installing || registration.waiting;
          if (!sw) { resolve(); return; }
          sw.addEventListener('statechange', function handler() {
            if (sw.state === 'activated') {
              sw.removeEventListener('statechange', handler);
              resolve();
            }
          });
        });
      }

      // 新規 Subscription を作成
      const applicationServerKey = urlBase64ToUint8Array(vapidPublicKey);
      const subscription = await registration.pushManager.subscribe({
        userVisibleOnly: true,
        applicationServerKey: applicationServerKey,
      });

      return JSON.stringify(subscription);
    } catch (e) {
      console.error('[WebPush] error:', e);
      return null;
    }
  };

  function urlBase64ToUint8Array(base64String) {
    const padding     = '='.repeat((4 - base64String.length % 4) % 4);
    const base64      = (base64String + padding).replace(/-/g, '+').replace(/_/g, '/');
    const rawData     = window.atob(base64);
    const outputArray = new Uint8Array(rawData.length);
    for (let i = 0; i < rawData.length; ++i) {
      outputArray[i] = rawData.charCodeAt(i);
    }
    return outputArray;
  }
</script>

11 fcm_service_web.dart を作成（Web 向け実装）

lib/services/fcm_service_web.dart
import 'dart:convert';
import 'dart:js_interop';

import 'package:flutter/foundation.dart';
import 'package:http/http.dart' as http;

// index.html で定義した window.subscribeWebPush() を Dart から呼び出す
@JS('subscribeWebPush')
external JSPromise<JSAny?> _subscribeWebPushJs(JSString vapidKey);

class FcmService {
  static Future<void> registerToken({required String userId}) async {
    if (!kIsWeb) return;
    try {
      // ① Laravel から VAPID 公開鍵を取得
      final Uri keyUri = Uri.https('（ドメイン）', '/api/vapid-public-key');
      final http.Response keyResponse = await http.get(keyUri);
      if (keyResponse.statusCode != 200) return;

      final Map<String, dynamic> keyData =
          jsonDecode(keyResponse.body) as Map<String, dynamic>;
      final String vapidPublicKey = keyData['public_key'] as String;

      // ② ブラウザの Web Push を購読（index.html の JS 関数を呼ぶ）
      final JSAny? jsResult =
          await _subscribeWebPushJs(vapidPublicKey.toJS).toDart;
      if (jsResult == null) return;

      final Map<String, dynamic> subscription =
          jsonDecode(jsResult.dartify()! as String) as Map<String, dynamic>;
      final String endpoint = subscription['endpoint'] as String;
      final Map<String, dynamic> keys =
          subscription['keys'] as Map<String, dynamic>;

      // ③ Laravel に Subscription 情報を送信して DB に保存させる
      final Uri subUri = Uri.https('（ドメイン）', '/api/web-push/subscribe');
      await http.post(
        subUri,
        headers: <String, String>{
          'content-type': 'application/json',
          'Accept': 'application/json',
        },
        body: jsonEncode(<String, String>{
          'user_id': userId,
          'endpoint': endpoint,
          'p256dh': keys['p256dh'] as String,
          'auth': keys['auth'] as String,
        }),
      );
    } catch (e) {
      debugPrint('WebPush registerToken error: $e');
    }
  }
}

13 main.dart でアプリ起動時に登録する

ログイン済みのままアプリを再起動した場合も subscription を自動復旧させるために、起動時にも registerToken を呼ぶ。

lib/main.dart（関係する部分のみ）
import 'dart:async';
import 'services/fcm_service.dart';

Future<void> main() async {
  WidgetsFlutterBinding.ensureInitialized();

  final SharedPreferences prefs = await SharedPreferences.getInstance();
  final String loggedInUserId = (prefs.getString('loggedInUserId') ?? '').trim();

  // 起動時にログイン済みなら push subscription を再登録する。
  // DB から subscription が消えていても（410 Gone 後など）自動で復旧する。
  // unawaited で fire-and-forget にして起動を遅延させない。
  if (loggedInUserId.isNotEmpty) {
    unawaited(FcmService.registerToken(userId: loggedInUserId));
  }

  // ... 以降は通常の runApp など ...
}

8. トラブルシュート

症状原因対処
スマホに届かない（PC には届く） push-sw.js が Flutter SW と同じスコープで競合し、waiting のまま activated にならない index.html の scope を /blue_sky/push/ に変更してデプロイ。スマホでキャッシュクリア後にアプリ起動
ログインし直したら届かなくなった 毎回 unsubscribe→再登録 していたため、再登録失敗時に DB が無効なエンドポイントのまま 既存 subscription を unsubscribe せず再利用するコードになっているか確認
起動直後は届かない（ログインし直すと届く） main.dart で起動時の registerToken が呼ばれていない loggedInUserId.isNotEmpty 時に unawaited(FcmService.registerToken(...)) があるか確認
DB のレコードが増え続ける キャッシュクリアのたびに endpoint が変わり新しいレコードが INSERT される 正常動作。古い endpoint は次回の push 送信時に 410 Gone → 自動 DELETE される
Mac の集中モード中に届かない macOS の集中モードが通知をブロックしている 集中モードを OFF、またはシステム設定 → 通知 → Chrome → 通知を許可
iPhone Safari で届かない Safari の Web Push は PWA（ホーム画面に追加）が必要 Safari でサイトを開き「共有 → ホーム画面に追加」してから通知を許可する

Claude Code へ — 次回実装するときの注意点

  • 赤文字（BlueSky 系）を実際のプロジェクト名に置き換えること。
    PascalCase（クラス名）・camelCase（変数・関数名）・snake_case（DB・URL・ファイル名）を使い分ける。
  • スコープ（/blue_sky/push/）を一度決めたら変更しないこと。
    途中で変えると既存ユーザーの subscription が無効になり、全員にキャッシュクリアを促す必要が生じる。
  • 既存 subscription を unsubscribe() しないこと。
    getSubscription() で既存があればそのまま返す実装が正しい。
  • 410 Gone は正常な自動削除のサイン。
    WebPushService が自動で DELETE するので手動対応は不要。
  • Firebase を使わないこと。
    パッケージは Composer の minishlink/web-push のみ。Flutter 側に firebase 系パッケージは一切不要。
  • push-sw.js は web/ フォルダ直下に置くこと。
    Flutter のビルドで build/web/push-sw.js として出力され、VPS 上の公開パスに配置される。
